   <h3>¿Qué es Tailwind CSS <img src="https://raw.githubusercontent.com/devicons/devicon/6910f0503efdd315c8f9b858234310c06e04d9c0/icons/tailwindcss/tailwindcss-original.svg" height="15" />?</h3>


   <p><strong>Tailwind CSS</strong> es un <i>framework CSS</i> de código abierto que se destaca por su enfoque único de diseño utilitario. En lugar de proporcionar estilos preestablecidos para elementos específicos, proporciona una serie de clases utilitarias que se pueden aplicar directamente a los elementos HTML para estilizarlos. Esto simplifica el proceso de creación de estilos para páginas web y aumenta la productividad.</p>

   <p>Trabaja con clases, que se van sumando con cada clase del proyecto hasta formar la página.</p>

   <p><strong>Tailwind CSS</strong> está diseñado con un enfoque <code>"mobile-first"</code>. Esto significa que las clases de utilidad sin prefijo <i>(como uppercase)</i> se aplican en todos los tamaños de pantalla, mientras que las clases de utilidad con prefijo <i>(como md:uppercase)</i> solo se aplican en el breakpoint especificado y superior. Por lo tanto, puedes construir fácilmente interfaces de usuario adaptativas sin salir nunca de tu HTML. Esto es similar a lo que puedes estar acostumbrado en otros frameworks como Bootstrap.</p>

   <br>

   <h3>¿Qué es el <i>package.json</i>?</h3>

   <p>Cuando ejecutas el comando <code>npm init -y</code> en la terminal de tu proyecto, npm (Node Package Manager) crea un archivo llamado <strong>package.json</strong> en el directorio actual del proyecto. El <strong>package.json</strong> es un archivo de metadatos para tu proyecto Node.js. Contiene información sobre el proyecto, como el nombre, la versión, la descripción, las dependencias, los scripts de ejecución y otras configuraciones importantes.</p>

   <p>La opción <code>-y</code> es una abreviatura de <code>--yes</code>, que hace que npm acepte automáticamente todas las preguntas realizadas durante la inicialización del <strong>package.json</strong>, utilizando los valores predeterminados para las respuestas. Esto es útil cuando deseas crear rápidamente un <strong>package.json</strong> con las configuraciones predeterminadas y no necesitas revisar o cambiar manualmente cada una de ellas.</p>

   <p>Al ejecutar <code>npm init -y</code>, npm crea un <strong>package.json</strong> con valores predeterminados, incluida la información mínima como el nombre del proyecto, la versión, la descripción y el punto de entrada (generalmente el archivo principal de tu aplicación). Luego puedes editar este archivo manualmente o agregar más información según sea necesario, como dependencias, scripts de ejecución, autor, licencia, entre otros detalles del proyecto.</p>

   <p>Un archivo <code>.json</code> es un tipo de archivo de texto utilizado para almacenar y transmitir datos de forma estructurada. Utiliza una sintaxis simple de pares clave-valor y se utiliza comúnmente en programación para configurar aplicaciones, intercambiar datos entre servidores y clientes web, y otras tareas de gestión de datos.</p>

   <br>

   <h3>¿Qué es el <i>deploy</i>?</h3>

   <p>Usado principalmente en el contexto de desarrollo de software, se refiere al proceso de lanzar o poner a disposición una aplicación o una nueva versión de un software para su uso en un entorno operativo, como un servidor o una plataforma en la nube. Esto generalmente implica la transferencia de los archivos y recursos necesarios al entorno de producción y la configuración adecuada para que el software funcione correctamente. El objetivo del deploy es hacer que el software sea accesible y utilizable por los usuarios finales.</p>

   <p>En programación, el término <strong>deploy</strong> está estrechamente relacionado con su traducción literal del inglés: <i>implantar</i>. En términos prácticos, hacer un deploy significa poner en marcha una aplicación que ha completado su desarrollo.</p>

   <p>Las etapas de un <strong>deploy</strong> son: 
      <p><code>Desarrollo</code> — la aplicación está solo en la máquina del programador, un entorno de prueba y error, donde el profesional tiene la libertad para crear y trabajar. Tanto el profesional como el código maduran para la próxima etapa,</p> 
      <p><code>Staging/Prueba</code> — también llamado <i>entorno de homologación</i>. La aplicación pasa por el primer deploy para estar disponible para el equipo de prueba, siendo accesible en una <i>intranet</i> o en un entorno restringido, y</p> 
      <p><code>Producción</code> — después de las pruebas, mejoras y correcciones, la aplicación se <strong>implanta</strong> en el entorno de producción, ahora disponible para los usuarios finales.</p>
   </p>

   <p>En resumen, el <strong>deploy</strong> es el proceso que permite que

 una aplicación salga del entorno de desarrollo y se ponga en uso, garantizando que los usuarios puedan interactuar con ella. Cuando creas un sitio web, un software o una aplicación y lo pones a disposición de los usuarios finales, estás realizando un <strong>deploy</strong>.</p>

   <h3>¿Qué es el método 'Splice' en <i>JavaScript</i>?</h3>

   <p>El método <strong>splice()</strong> en JavaScript es un método integrado para manipular arrays. Modifica el array original, eliminando, reemplazando o agregando elementos. La sintaxis del método splice() es la siguiente: <code>array.splice(indice, deleteCount, elemento1, ..., elementoN)</code></p>

   <p>Los parámetros son:
      <ul>
         <li><code>indice</code> : El índice en el que debe comenzar la modificación del array. Si es mayor que el tamaño total del array, no se modificará ningún elemento. Si es negativo, comenzará desde ese número de elementos desde el final.</li>
         <br>
         <li><code>deleteCount</code> : Un entero que indica el número de elementos antiguos que deben eliminarse. Si no se especifica, o si es mayor que el número de elementos restantes en el array comenzando por el índice, entonces se eliminarán todos los elementos hasta el final del array. Si es 0, no se elimina ningún elemento</li>
         <br>
         <li><code>elemento1, ..., elementoN</code> : Los elementos que se agregarán al array. Si no especificas ningún elemento, splice simplemente eliminará elementos del array.</li>
      </ul>
   </p>

   <p>El valor de retorno es un nuevo array que contiene los elementos eliminados. Si no se elimina ningún elemento, se devuelve un array vacío.</p>

   <p>Este es un ejemplo de cómo usar el método <strong>splice()</strong>:
      <br>
      <code>var myFish = ["angel", "clown", "mandarin", "surgeon"];</code>
      <p>// eliminar 0 elementos desde el índice 2 e insertar "drum"</p>
      <code>var removed = myFish.splice(2, 0, "drum");</code>
      <p>// myFish es ["angel", "clown", "drum", "mandarin", "surgeon"]
         <br>
         <p>// removed es [], ningún elemento eliminado</p>
      </p>
   </p>

   <h2>Observaciones y Notas para Estudio</h2>

   <ul>
      <li>Cuando realices un <code>npm init -y</code> en la Terminal para crear un <strong>package.json</strong>, el nombre del directorio de tu proyecto no puede tener más de una palabra separada, como <i>Menú en línea</i>, asegúrate de poner solo una palabra o, si tienes más de una, ponerlas juntas (de lo contrario, podría producirse un error al crear el paquete .JSON).</li>
      <br>
      <li>Dentro del archivo <strong>tailwind.config.js</strong>, en la parte <code>'content'</code>, este camino escrito que contiene HTML y JS, indicando que los archivos dentro de nuestra carpeta que terminan con estos dos, pueden usar Tailwind CSS.</li>
      <br>
      <li>En el archivo <strong>package.json</strong>, dentro de <i>scripts</i>, siempre configura para usar Tailwind CSS: <code>"dev": "npx tailwindcss -i ./styles/style.css -o ./styles/output.css --watch"</code></li>
      <br>
      <li>Cuando vuelvas a abrir tu proyecto con Tailwind CSS, no olvides configurar en el CMD (Símbolo del Sistema) el siguiente comando: <code>npm run dev</code> y abrir tu archivo de página web con "Abrir con Live Server".</li>
      <br>
      <li><code>md, lg, sm: medium | large | small</code> — <strong>Media Query:</strong> un recurso de CSS3 que permite adaptar el diseño de una página web a diferentes tamaños de pantalla y tipos de medios. Consiste en un tipo de medio y una o más expresiones que limitan el alcance de las hojas de estilo utilizando características de medios como ancho, alto y color. Esto facilita la creación de sitios web responsivos que se adaptan automáticamente al dispositivo del usuario.</li>
      <br>
      <li>La etiqueta <code>main</code> de HTML define el contenido principal dentro del <code><body></code> de un documento o aplicación. El contenido dentro del elemento <code>main</code> debe ser único para el documento y estar directamente relacionado con el tema central de la página o la funcionalidad central de la aplicación. Esto significa que las secciones de la página que son comunes a todo el sitio o aplicación, como los mecanismos de navegación, la información de copyright, el logotipo y los campos de búsqueda, no deben incluirse dentro del elemento <code>main</code>. Además, la etiqueta <code>main</code> no puede ser hija de los elementos <code>article</code>, <code>aside</code>, <code>footer</code>, <code>header</code> o <code>nav</code>. Debes usar la etiqueta <code>main</code> de HTML cuando necesites un elemento para insertar el contenido principal de la página. El contenido principal son los textos, imágenes, tablas que están relacionados con el tema propuesto del enlace en el que entramos. Este contenido debe ser único en la página, es decir, no debe repetirse a lo largo de la página.</li>
      <br>
      <li><strong>Propiedad <code>flex</code> en CSS</strong> — utilizada en un contenedor flexible para controlar el comportamiento de sus elementos secundarios. Es una abreviatura de las propiedades <i>flex-grow</i>, <i>flex-shrink</i> y <i>flex-basis</i>, que controlan cómo un elemento crece, se encoge y se dimensiona inicialmente dentro del contenedor.</li>
      <br>
      <li><strong>Propiedad <code>grid</code> en CSS</strong> — utilizada para crear un diseño de cuadrícula bidimensional. Es una abreviatura de varias propiedades que definen las filas y columnas de la cuadrícula, así como las áreas de la cuadrícula. Esto permite un control preciso sobre la posición y el tamaño de los elementos en la página.</li>
      <br>
      <a href="https://medium.com/@sarahcalande/flexbox-vs-css-grids-ba5d75287dd6"><img src="https://miro.medium.com/v2/resize:fit:871/1*kI78qZQjw_ax1kJo4lPOcw.png" alt="Créditos: https://www.youtube.com/watch?v=RSIclWvNTdQ" /></a>
      <br>
      <br>
      <li><strong>Método <code>addEventListener()</code> en JavaScript:</strong> utilizado para adjuntar un controlador de eventos a un elemento del DOM (Documento de Objeto de Modelo). Permite especificar una función que se ejecutará cuando ocurra un evento específico en el elemento. El <code>addEventListener()</code> escucha los eventos en los elementos y ejecuta acciones específicas cuando ocurren esos eventos. Es una herramienta poderosa para la interactividad y la manipulación de elementos en la web.</li>
      <br>
      <li><code>Slice</code> en <strong>JavaScript</strong>: se refiere a un método de matriz que devuelve una copia de parte de una matriz original. No altera la matriz original, sino que crea una nueva matriz que contiene los elementos especificados. Un "slice" es una parte de una matriz. Es como cortar una rebanada de pastel. Imagina que tienes un pastel (que sería tu matriz) y quieres una parte específica de él, como solo la parte central. El método <code>slice()</code> hace exactamente eso para las matrices.. No altera la matriz original, pero <strong>crea una nueva matriz</strong> que contiene los elementos especificados. El método <code>slice()</code> acepta <strong>dos argumentos opcionales</strong>: el índice inicial y el índice final del rango que se copiará. Si no se proporciona ningún argumento, <code>slice()</code> copiará todos los elementos de la matriz original.</li>
      <br>
      <li><strong>innerHTML, innerText y TextContent en JavaScript</strong> : propiedades utilizadas para manipular el contenido de un elemento HTML, pero funcionan de manera ligeramente diferente. El <code>innerHTML</code> devuelve o define el contenido HTML de un elemento. Incluye tanto el texto como las etiquetas HTML dentro del elemento. Por otro lado, el <code>innerText</code> devuelve o define solo el texto "visible" de un elemento. Ignora las etiquetas HTML y considera el estilo CSS (como display: none o visibility: hidden). Y el <code>textContent</code> devuelve o define todo el contenido de texto de un elemento, incluidos los nodos de texto en elementos secundarios. Ignora las etiquetas HTML, pero no considera el estilo CSS.</li>
      <br>
      <li><strong>¿Qué es una <code>API</code> (Application Programming Interface — Interfaz de Programación de Aplicaciones)</strong>: actúa como un intermediario que permite la comunicación entre diferentes programas informáticos, facilitando el intercambio de información y servicios. Por ejemplo, en una aplicación de viajes, la API permite que la aplicación envíe solicitudes a varias aerolíneas y reciba opciones de vuelo en respuesta. Esto es posible gracias a la definición de un "idioma común" por parte de la API, que permite el intercambio de datos sin necesidad de comprender internamente el funcionamiento de cada sistema. Las pruebas en las API son esenciales para garantizar su funcionalidad, seguridad, rendimiento y usabilidad, asegurando así la entrega de productos de software robustos y de alta calidad.</li>
      <br>
      <li><strong>Toastify JS</code> para alertas/notificaciones</strong> : es una biblioteca de notificaciones de <strong>toast</strong> — <i>notificaciones de computadora con un mensaje ligero que aparece en la interfaz del usuario y desaparece automáticamente después de un corto período de tiempo. Se llama 'toast' porque, al igual que un trozo de pan que salta de la tostadora, la notificación aparece y desaparece sin interrumpir la actividad del usuario. Generalmente, se utiliza para proporcionar comentarios rápidos sobre una operación sin requerir una acción del usuario. Por ejemplo, puede informar que se ha enviado un mensaje o que se ha guardado un archivo</i> — liviana con JavaScript. Aquí hay algunas de sus características: notificaciones múltiples y apiladas, no bloquea el hilo de ejecución, opciones de personalización que incluyen texto de notificación, duración, color de fondo de toast, visualización del ícono de cierre, posición de visualización y posición de desplazamiento.</li>
      <br>
   </ul>
